# EA-010 — Constitutional Knowledge Library (P3)

**The Knowledge Library is a generated view, not a hand-maintained catalog.** Every constitutional artifact is already self-describing (STD-003) and inventoried (MAP-001); the Library is CBS-001's compilation of that metadata into a browsable, CRID-keyed catalog of Knowledge Objects. **Not redesigned — populated from MAP-001.** Lifecycle: Architecture Draft (built as a generated view; awaiting Founder ratification as the canonical EA).

## Library structure (sections)
Books · Volumes · Architecture · Standards · Doctrines · ADRs · Policies · Registries · Recovery · Validation · Roadmaps · Historical releases · Generated indexes. Every entry references **CRIDs**, never filenames.

## Knowledge Object (per artifact)
Each object is one artifact's compiled record, drawn entirely from its front-matter + MAP-001:
`metadata · relationships (8 declared relations) · summary · authority (canonical_owner + ratification_authority) · dependencies · lifecycle_state · version · validation (validated_by) · cross-references (related_artifacts)`.

## Realization (what exists today)
- **`constitution/index/knowledge_index.json`** — the generated Library view (all artifacts as Knowledge Objects), emitted by CBS-001.
- Section indexes (`ea_index.json`, `standard_index.json`, `doctrine_index.json`, `registry_index.json`, `adr_index.json`, `recovery_index.json`, `validation_index.json`, `roadmap_index.json`) are the Library's shelves.
- Relationships come from `dependency_graph.json`.
- **Books / Volumes** (`BOOK-*`, `VOL-*`) are reserved for future knowledge *content* (e.g. legal knowledge volumes); none exist yet — the shelf is defined, empty.

## Governance
Governed by EA-000; generated by CBS-001; validated by CBS-001 (metadata/reference integrity) + CA-001. The Library never becomes a second source of truth — it is compiled from artifact metadata and regenerated each release. Ratifying EA-010 promotes this generated view to the canonical Knowledge Library.
