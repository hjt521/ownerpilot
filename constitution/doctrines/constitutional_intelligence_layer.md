# Doctrine — The Constitutional Intelligence Layer

**Status:** ADOPTED (Founder directive, 2026-07-24) · **Scope:** every intelligence model in the constitution, present and future.

## Founder directive (verbatim intent)
> Do not accelerate feature development. Continue strengthening constitutional foundations. Every new intelligence model (Behavioral, Trust, Decision, Confidence, Negotiation, etc.) should be designed as a governed constitutional capability that shares a common evidence model, traceability model, and review model. Avoid creating independent scoring systems or isolated frameworks.

## The principle
The strategic objective is a **Constitutional Intelligence Layer**: a shared foundation so that every intelligence model operates from **one evidence model, one governance model, and one traceability model** — rather than each model evolving its own. This is what prevents OwnerPilot from becoming a collection of disconnected AI modules.

## Binding rules for every intelligence model
1. **Governed constitutional capability.** Each model is registered and governed as a `constitution` capability (audit + extend the existing registries) — never a bolt-on framework outside the constitution.
2. **Shared evidence model.** All models classify evidence with the *same* vocabulary (direct-verified, first/third-party statement, system-generated, corroborated, contradictory, missing, inference, AI-generated). AI interpretation is never represented as source evidence.
3. **Shared traceability model.** Every conclusion traces to its evidence, its assessor identity (`approved_by`/assessor is a logical governance identity), its model version, and what it supersedes. The Knowledge Graph (P5.5) is the connective tissue for this traceability.
4. **Shared review model.** CA-001 is the common assurance path: it audits governance and evidence integrity across all models. AI output is never ratification; missing evidence yields `Indeterminate`.
5. **No independent scoring systems.** No model invents its own isolated score, hidden adverse scoring, or universal cross-context number. Distinct questions stay distinct — Behavioral (*what is likely*), Trust (*permissible reliance*), Confidence (*evidence completeness/quality*), Negotiation (*engagement strategy*), Decision (*compare actions*).
6. **Foundations before features.** Strengthen the shared foundation before adding model capability. Sequencing (canonical inventory → knowledge library → metadata → capability registry → knowledge graph → models) reflects this deliberately.

## Model map (current program)
- Behavioral Intelligence — predicts likely behavior.
- **TM-001 Trust Model** (PROPOSED, P6) — permissible reliance.
- **CM-001 Confidence Model** (PROPOSED, after TM-001) — evidence completeness/quality.
- Negotiation Intelligence — engagement strategy.
- Decision Intelligence — compares actions.
- **CA-001 Constitutional Auditor** (ratified) — audits governance + evidence integrity across all of the above.

Each shares the layer's evidence/traceability/review foundation. Adding a model means extending the shared foundation, not forking it.
